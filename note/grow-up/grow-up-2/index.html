向上生长 二

发表于 2022-05-03 更新于 2022-05-04 分类于 阅读笔记 ， 向上生长 本文字数： 1.3k 阅读时长 ≈ 2 分钟

        这是阅读《向上生长》后的第二篇阅读笔记。究竟为什么要写这些看似没用的东西？也许是在借鉴作者“九边”的做法吧。他选择将收获的新知识写下来发一条微博，我则选择将阅读这本书每一章后所带给我的感觉记录在自己的 Blog 中。

        我并非与九边相同的前端工程师，对于前端的相关语言也只是略懂一二而已，不过这并不影响我写这篇阅读笔记的进程。“对之前读过的书有个记录，在吸收新知识的同时锻炼自己”。我从来没有想过还可以这样。对此，我真的要感谢九边教会我的这些。

        正如同九边所说，读书有用，他将读书比喻成了搜索引擎。这样的比喻很直观，很形象。搜索引擎只是工具，它是用来帮助人们学习而非取代人们学习。人就像是木匠，搜索引擎就是工具，而知识则相当于木材。只有工具而没有木材什么也做不成，只有木材没用工具也是什么也无法做到；只有工具和木材而没有木匠更是不必多说。

        又想起了曾经不知道在哪里听到的话，只要你想做，就不会缺少做这件事的时间。你有没有时间去读书纯粹是由你是否想去读书而决定的。因为你想去读，所以才会将时间用在这上面；如果你不想去读，那你怎么也不想将时间浪费在这上面。我与九边的观点是不同的，我认为没有人是懒惰的，每个人都是十分勤快的，他只是不想将时间浪费在这上面而已。而所谓浪费，则是你将时间用在了不想做的事情上。

        每个人都是不了解自己的，嘴上说着你不了解我之类的话，其实连自己也不了解自己。由这个思路来讲，其实每个人都不应该说“你并不了解我”之类的话。

        我与九边不一样，九边懂得要比我多得多。他会用神经学来解释习惯，而我只能用条件反射来解释。人在不适应某种习惯时会十分的难受，我认为这也是条件反射所导致的。他会用雪球理论来解释日积月累，而我只会用最简单的降雨来解释。

        在前面说过要注重数量，但其实单单注重数量是不行的。就拿读书来举例，你在一年中读完十本书，而我在一年中仅读完半本书，这有区别吗？有区别，且不单单时数量的区别。一年中读完10本书这确实符合上文所说的注重数量，但只在乎读书的数量就不去理解、消化书中所蕴含的知识和道理了吗？我认为这是不行的。他在一年中读完十本书但是并没有消化书中的知识，我在一年中只读完半本书但我将书中的知识消化完全了。读书的意义不在于读，而是在于理解。只有将书中的知识完全消化之后才可以被称之为读书。

        完全消化这在实际中并不可能，而我们所需要做的则是将书中的知识消化到 70% 及以上便可以了。

        在这里我似乎要反驳一下之前的观点了。以写小说为例子，有些人会因为打字速度太慢而放弃，如果按照九边作者所说，多写几年，打字速度一定会提升上去。其实这并非是结果，另一个结果可能是打字速度并没有提升多少。再结合上述读书的例子，这时我便对九边所说的话产生了怀疑。量的提升有可能并不会带来质的飞跃，由量变到质变的前提是从量变中学到些什么。打字速度由慢到快是自己在打字这方面上摸到了门道，在打字这一过程中学到了什么更为快速、简单的方法来提升自己的打字速度。

        换句话来总结上述则为在注重的同时还要在量变的过程中学到些什么技巧，否则你在这一过程中所做的纯属是浪费时间。
