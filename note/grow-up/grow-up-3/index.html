向上生长 三

发表于 2022-05-04 分类于 阅读笔记 ， 向上生长 本文字数： 1.8k 阅读时长 ≈ 3 分钟

        这是阅读《向上生长》后的第三篇阅读笔记。这一章也许是最难懂的一章吧，读了很多遍没有读懂这一章究竟是什么意思。不过既然九边在这一章中提到了工具，那我们不妨就来讨论一下工具对于我们的学习有哪些重要作用吧。

        九边将工具分为了两种——简单的工具与复杂的工具。简单的工具可以指木棍等简单的物品，复杂的工具就可以指 C4D、Blender、Adobe Photoshop、Ableton Live 11 还有之前有提到过的 Eclipse 与 JetBrains 的很多产品。无论简单还是复杂，它们都可以用来辅助人的创造。当然，只有工具是不够的，我们还需要使用工具的人。人在这里是一个很大的概念。

        动物为什么不能使用工具、不能使用语言、不能创造文字？那是因为没有人脑。所有的动物都有大脑，但并非所有的动物都有人脑。人脑是人类所特有的器官。人类所作出的任何动作都与人脑有关。所以若说的精细些，在使用工具时我们还需要人脑。它通过发出指令来使用工具，在使用工具的同时它又会将自己所学的记录在脑中。

        当然，有时候仅仅只有一个工具是不足的，我们还需要多个工具。以 Python 的爬虫为例，它不仅需要一个编辑代码的工具，还需要一个运行代码的工具；但只有这些是不行的，想要写好一个爬虫，它多多少少都需要用到一些“轮子”。“轮子”是什么？“轮子”就是“库”，它包含着一些码农已经写好的函数。

        爬虫可以分为三个步骤：请求、解析、存储，每个步骤都会用到不同的“轮子”。爬虫是模仿人类访问网页获取信息的一种工具。既然是模仿，那我们不妨想一下查找信息的几个步骤。我们不妨以找一张图片为例。首先我们需要与百度或者其他的搜索引擎建立 TCP 连接，TCP 链接建立完成后Web 浏览器会向 Web 服务器发送请求，然后 Web 服务器会给予 Web 浏览器答应，一般情况下 Web 服务器向浏览器发送了请求数据，他就会关闭 TCP 连接，此时便会在浏览器上显示我们所见到的网页了。（关于 TCP/IP 协议可以访问维基百科）第二步便是在网页上查找我们需要的图片，找到合适的图片后我们需要将它存储到本地计算机上。至此，一张图片便被我们成功的从网页上下载下来了。

        爬虫所做的，就是使用代码将上述行为模拟执行。在编辑代码是你需要一个编辑 Python 代码的工具，代码写完之后还需要一个执行代码的工具，你安装工具还需要一个系统环境。当然，也有一种工具可以执行代码也可以编写代码，比如 JetBrains 的 PyCharm 。关于编写的代码，在请求 HTTP 时，我们经常需要用到 urllib 、requests 等库；请求过后我们需要解析网页中我们所需要的信息，在解析网页信息时，我们经常需要用到 Beautifulsoup 、lxml 等库；提取了所需要的信息过后便需要将解析的信息存储下来，在存储所解析后的信息时，我们会用到 pymysql 。

        这时你会发现，两种不同的工具简易程度并不一样。自己从网站上保存一张图片只需要

短短几步，而自己编写网络爬虫则需要去了解一堆东西。比如什么是 TCP/IP 协议族 ，较为基础的 Python 语法，网页的数据结构等等。由此可见，简单的工具并不能带给我们多少突破，而复杂的工具将会教给我们很多东西。同时，工具的复杂程度不同，我们所需要使用工具的时间、理解工具原理的时间也就不同、工具的便捷程度也不相同。

        接下来便是时间，关于时间这方面我与九边的观点还是不大相同。时间长收获的东西并不一定就多，不断的投入没有回报的时间只是单纯的浪费而已。正如第二篇所说，你所做的仅仅只是读书，但没有“读书”。书不在多，精读则灵 ，读书的很多，但用心去读一本书的人很少。所投入的时间越长，所理解的可能就越深，但这一切的前提都是在这些时间上有一个总体的规划。在什么时间应该做什么，应该学会什么，这一点我想是十分重要的。

        一个宅在家里打游戏的阿宅和一些一天都坐在电脑前工作的程序员有什么相同的地方？没错，他们都不经常运动。还记得曾经看到的一个新闻，一个人在网吧连续打了几天游戏之后猝死。这是缺乏运动的表现。人的身体就如同一块电池，运动则是为这块电池充电。还记得曾经有人给我说过脑内运动也是运动。没错，大脑无时无刻在记录一些东西或者执行某条命令，这样看来大脑确实在运动，而且是无时无刻的运动。为什么会这样？因为它知道，当它不再运动的时候，它也就死了。不过大脑在运动只是为了能使“大脑”这个器官能活下去，心脏在跳动也只是为了“心脏”这个器官能够活下去，同样的，你在运动你也只是为了你自己能够活下去。
