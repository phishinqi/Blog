    关于我搭建博客的经验

    发表于 2022-05-04 分类于 教程 本文字数： 2.1k 阅读时长 ≈ 4 分钟

    0x0.前言

            已经忘了当初为什么要搭建博客，可能只是为了和周围的同学装一下吧，不过在现在看来，博客正在改变着我的生活，正如前面某篇文章所言——“若你不想在身消体灭之际就被世界所遗忘，那就写一些值得一写的东西，或去做一些值得一写的事情”。虽然不指望着被记录下来，但选择了这条路就坚持走下去，这样做是不会有坏处的，它是会让我懂的东西越来越多。

    0x1 Hexo

            Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。你可以将 Hexo 部署在 Github Pages 或 Heroku 等平台。

    0x2 Node.js

            Node.js 是一个开源并跨平台的 JavaScript 运行环境。Node.js 为用户提供了大量的库供各位程序员自由使用。

    0x3 Markdown

            Markdown 是一种轻量级标记语言，也可能是世界上最简单的标记语言。它允许人们使用易读易写的纯文本格式编写文档，然后将其转化为有效的 HTML 或 XHML 文档。

    0x1.准备

            在搭建博客前需要做好充足的准备，下方则是我搭建该博客所用到的一些东西：

    1. Node.js（npm），用于安装 Hexo

    2. Hexo 静态博客框架

    3. NexT 博客所用主题，可选用其他主题

    4. Github 用于托管，也可使用 Gitee ，Gitee 的操作与 Github 类似

    5. Git Bash客户端

    6. Clash For Windows 用于代理，以防止 Github 无法访问.可选用 Trojan、V2ray、SSR 等

    4x1.Hexo 设置

            Hexo 配置文件为博客根目录下的 _config.yml

    一些常用设置

    原名 译名
    title 标题
    subtitle 副标题
    anthor 作者
    language 语言
    url 网址，因为使用 Github Pages 所以将其设置为 https://{username}.github.io
    deploy 因为要推送到 Github，所以将 type 的值改为 ‘git’，将 repo 的值改为仓库链接

    4x2.NexT 设置

            对于 NexT 的配置文件可从官网查看。
