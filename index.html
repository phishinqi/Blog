关于如何正确理解 Eclipse 与 IDEA

发表于 2022-05-04 更新于 2022-05-03 分类于 文章 本文字数： 1.5k 阅读时长 ≈ 3 分钟

0x0.前言

        相信很多程序员都知道两款较为普及的 Java 集成开发环境（IDE）吧—— Eclipse 与 IntelliJ IDEA。

        先说说 Eclipse 吧，首先，它最初发布于 2001 年 11 月 7 日。Eclipse 由 Java 编写，同样，它的主要用途也是开发 Java 程序。当然，你还可以使用 Eclipse 来进行 C、C#、Cpp、Python 等语言的开发。要说 Eclipse 最亮眼的地方，莫过于它是开源且免费的，它还可以添加一些实用的拓展来辅助各位码农的开发。

        再来谈谈 IntelliJ IDEA （下文简称IDEA） 吧，相对于 Eclipse 来讲 IDEA 分为两个版本—— Ultimate 与 Community。Community 虽然免费，但它的限制仍有很多。但虽说一个版本付费一个版本免费，二者却都可用于商业开发。同时，JetBrains 对于学生的优惠也算是下了血本。它最初发布于 2001 年 1 月。与 Eclipse 相同的是 IDEA 也有 Java 语言编写，也主要用于 Java 程序的开发。你还可以使用 IDEA 编写 HTML、Kotlin、R、PHP、Go 等语言。

0x1.区别

1x1.内存占用

        从内存占用来讲，IDEA 要比 Eclipse 多得多。这可能对于有 32 GB RAM 或以上的电脑可能影响并不大，但对于仅有 8 GB RAM 的电脑来讲可能有些吃力了。不过仅仅是看 UI 界面也能发现为什么 IDEA 占用更多了吧？

1x2.主题

        要比起 IDE 自带的 主题，那可就真的是一个天上一个地下了，IDEA 的界面十分美观，有多种主题可以挑选。而 Eclipse 在这方面似乎就有些不足了。在 Eclipse 中仅有黑色和白色主题，若想为 Eclipse 更换主题，还需要安装一个 devstyle 插件。IDEA 还内置了自己设计的、专为程序员设计的字体—— JetBrains Mono。它可以更好的区分 “1” 、“I”、“l” 以及 “0”、“O” 等等。

1x3.自动补全

        在开发的过程中，我们可以使用自动补全来节省一个字母一个字母去手动输入的时间。而自动补全并非简简单单的将这条所需要的代码补全就可以了，它还需要联系上下文。在这一点中确实要夸赞一下 IDEA 了，IDEA 无需去按一些无关紧要的按键便可以为你提供一种或多种可供选择的方案。而在 Eclipse 中，你需要按下 Ctrl 与 Space 才可以查看到很多被标记高亮的代码和一些与你的开发并无关系的各种信息。

0x2.各自的优点

2x1.IDEA

        IDEA 为用户提供了很多默认且可供选择的功能。
        字太小看不清？字太多密密麻麻的很不舒服？试试通过鼠标滚轮来改变字体大小吧。

2x2.Eclipse

        在 Eclipse 中，你可以通过双击标签或 Ctrl + M 来最大化控制台。

        在Eclipse 中，你只需要将鼠标悬停在代码上方便可以轻松查阅 Javadoc。

0x3.我应该选择什么 IDE ？

        从根本上来讲，两种 IDE 的功能性都基本相同，使用何种 IDE 进行开发都是自己的喜好，只有合适自己的 IDE 才是最好的、功能性最强的 IDE。如果硬要在两款IDE中选择一款，我会给予几条建议：

  1. 如果电脑配置并不怎么样，推荐使用 Eclipse。

  2. 如果是初学者，建议使用 Eclipse。

  3. 如果家里有矿/学生，可以购买 IntelliJ IDEA Ultimate。

  4. 如果注重美观，建议IntelliJ IDEA。

  5. 如果注重实用性，建议Eclipse

0x4.总结

        在当今时代，Eclipse 显得过于老古董了，对于现代人来说，任何事情都不想多花费一点无用的时间，而 IDEA 选择的将大部分功能集成在一起，确确实实的方便了人们很多。单看 IDEA 的话，JetBrains 似乎并不怎么起眼，但是请不要忘记，它还有 PyCharm 等专门为其他语言设计的编辑器或编译器。

注：本文将会讲解有关Minecraft Java服务器搭建的一些问题，其中包括Forge和Fabric服务器搭建，我将默认你所使用的为Windows操作系统，另外，本文不含插件服搭建。

服务端的选择与使用

Minecraft的服务端种类很多，那么我们应该如何去选择并使用它们呢？

原版

Minecraft原版服务端。原版服务端又被称为官服，它的特性与原版相一致，是完全没有经过任何修改的，十分适合基友服联机使用，此外，它在很多地方都有方法下载，这其中包括MCWiki。

下载

下面将演示Minecraft 1.17.1原版服务端的下载方法。

首先我们在MCWiki打开MC Java 1.17.1的词条，此时我们将目光转向右边。

我们会发现下载处包含服务端，我们点击它即可下载。

你也可以在官网下载。注：官网默认为最新游戏版本。

Mod

Forge

Forge服务端，顾名思义，Forge服务端即为Forge所推出的服务端，可以使用Forge Mod。

下载

下面我将演示Minecraft 1.16.5 Forge服务端的下载方法。

首先我们打开Forge的官网，找到目前最新版本

点击Installer即可下载。

注意：我们现在下载的并非服务端，而是Forge的安装文件，部分版本的安装方法可能有些不同，我将会在最后进行讲解

双击运行我们下载的安装文件。

选择Install server

然后选择一个空文件夹，等待安装即可。

Forge服务端安装时间可能较长，其中可能需要手动补全libraries库，我们可以通过查看log来找到所需文件手动下载。建议使用特殊上网方法进行安装，此处不在过多赘述。

我们点击Download installer，无论是.jar后缀还是.exe后缀都不会影响我们接下来的操作。

点击Server，同样选择一个空目录等待安装即可。

如果需要更换Minecraft版本，请点击下拉框选择。

向上生长 一

发表于 2022-05-03 分类于 阅读笔记 ， 向上生长 本文字数： 985 阅读时长 ≈ 2 分钟

        在2022的某一天，我意外的接触到了这本书——《向上生长》。从那时起，我便迷恋在这本书简洁的语言中。

        九边，这是个我从来没又听说过的名字。接触到他也仅仅是因为《向上生长》这本书。通过阅读这本书，我收获了很多。在书中，它教会了我如何去学习，学习的意义究竟是什么，我们为什么要学习。

        在书中，作者通过“热力学第二定律”解释了人为什么需要学习——获取持续输入的能量。不再学习的人就如同炉子里的火，没有能量的摄入结果只能是熄灭后化为灰烬。对于一个人来说，我们需要摄入的能量出了基本的营养外还有知识。我们可以通过挑战、学习来获取新的知识。

        知识可以分为两种，书本上存在的与书本上不存在或书本上不容易被注意、找到的。

        人生不能一路平坦，想要活下去我们必须要适应坎坷的道路，我们踏过这条坎坷的道路的过程可以理解为实践。不要怕，去挑战一下自己认为不可能的事情吧。这样才有新的知识摄入。

        但总是摄取自己所在的领域的知识是不够的，这样出去还是什么都不懂，因此我们需要摄取一些自己所在领域之外的知识。相信我，那总归不是无用的，总有一天他会用到的。

        有些读书的人会发现，自己长时间的阅读并没有收获太多，这其中的表现则可以形象的描述为自己的考试分数。不要担心，学习是一个循循渐进的过程，由量变从而产生质变。还记得在政治课本上的一句话：“量变是质变的基础，质变是量变的必然趋势”，有了量变才会在之后产生质变。一根木棍和一根铁棍的坚硬程度不同，这是因为“质”的不同。

        你可以将学习的知识比作一个水分子，一个水分子什么事情也无法做到，但成千上万乃至上亿个水分子呢？你吃东西的时候，食物所含有的能量会突然的传递给你吗？不会。因为吃进嘴里的食物有一个消化的过程。这并不是让你将书本上的字吃进肚子里让它消化，我相信也没有人会这样做。

        还记得小时候大人们常给我们讲过的有关于团结的故事吗？一根筷子很容易掰断，但一捆筷子则不容易被掰断。我们不妨想一下，从一根筷子到一捆筷子，这其中并没有“质”的变化，它仅仅是量的变化。团结又何尝不是如此？大家都是人，本质上并没有什么不同，所以团结的实质也只是量变所产生的结果而已。当你认为你的学习没有进展的时候，那请等一下吧，你所学习的知识会在之后的某一天突然“涌现”出来，这时，它们终于会派上了用场。当然，等待的过程并不是在亭子里坐着欣赏着自己也看不懂的东西，你可以在等待的时候学习新的知识。
