        Node.js 是一个开源并跨平台的 JavaScript 运行环境。Node.js 为用户提供了大量的库供各位程序员自由使用。

0x3 Markdown

关于如何正确理解 Eclipse 与 IDEA

发表于 2022-05-04 更新于 2022-05-03 分类于 文章 本文字数： 1.5k 阅读时长 ≈ 3 分钟

0x0.前言

        相信很多程序员都知道两款较为普及的 Java 集成开发环境（IDE）吧—— Eclipse 与 IntelliJ IDEA。

        先说说 Eclipse 吧，首先，它最初发布于 2001 年 11 月 7 日。Eclipse 由 Java 编写，同样，它的主要用途也是开发 Java 程序。当然，你还可以使用 Eclipse 来进行 C、C#、Cpp、Python 等语言的开发。要说 Eclipse 最亮眼的地方，莫过于它是开源且免费的，它还可以添加一些实用的拓展来辅助各位码农的开发。

        再来谈谈 IntelliJ IDEA （下文简称IDEA） 吧，相对于 Eclipse 来讲 IDEA 分为两个版本—— Ultimate 与 Community。Community 虽然免费，但它的限制仍有很多。但虽说一个版本付费一个版本免费，二者却都可用于商业开发。同时，JetBrains 对于学生的优惠也算是下了血本。它最初发布于 2001 年 1 月。与 Eclipse 相同的是 IDEA 也有 Java 语言编写，也主要用于 Java 程序的开发。你还可以使用 IDEA 编写 HTML、Kotlin、R、PHP、Go 等语言。

0x1.区别

1x1.内存占用

        从内存占用来讲，IDEA 要比 Eclipse 多得多。这可能对于有 32 GB RAM 或以上的电脑可能影响并不大，但对于仅有 8 GB RAM 的电脑来讲可能有些吃力了。不过仅仅是看 UI 界面也能发现为什么 IDEA 占用更多了吧？

1x2.主题

        要比起 IDE 自带的 主题，那可就真的是一个天上一个地下了，IDEA 的界面十分美观，有多种主题可以挑选。而 Eclipse 在这方面似乎就有些不足了。在 Eclipse 中仅有黑色和白色主题，若想为 Eclipse 更换主题，还需要安装一个 devstyle 插件。IDEA 还内置了自己设计的、专为程序员设计的字体—— JetBrains Mono。它可以更好的区分 “1” 、“I”、“l” 以及 “0”、“O” 等等。

1x3.自动补全

        在开发的过程中，我们可以使用自动补全来节省一个字母一个字母去手动输入的时间。而自动补全并非简简单单的将这条所需要的代码补全就可以了，它还需要联系上下文。在这一点中确实要夸赞一下 IDEA 了，IDEA 无需去按一些无关紧要的按键便可以为你提供一种或多种可供选择的方案。而在 Eclipse 中，你需要按下 Ctrl 与 Space 才可以查看到很多被标记高亮的代码和一些与你的开发并无关系的各种信息。

0x2.各自的优点

2x1.IDEA

        IDEA 为用户提供了很多默认且可供选择的功能。
        字太小看不清？字太多密密麻麻的很不舒服？试试通过鼠标滚轮来改变字体大小吧。

2x2.Eclipse

        在 Eclipse 中，你可以通过双击标签或 Ctrl + M 来最大化控制台。

        在Eclipse 中，你只需要将鼠标悬停在代码上方便可以轻松查阅 Javadoc。

0x3.我应该选择什么 IDE ？

        从根本上来讲，两种 IDE 的功能性都基本相同，使用何种 IDE 进行开发都是自己的喜好，只有合适自己的 IDE 才是最好的、功能性最强的 IDE。如果硬要在两款IDE中选择一款，我会给予几条建议：

  1. 如果电脑配置并不怎么样，推荐使用 Eclipse。

  2. 如果是初学者，建议使用 Eclipse。

  3. 如果家里有矿/学生，可以购买 IntelliJ IDEA Ultimate。

  4. 如果注重美观，建议IntelliJ IDEA。

  5. 如果注重实用性，建议Eclipse

0x4.总结

        在当今时代，Eclipse 显得过于老古董了，对于现代人来说，任何事情都不想多花费一点无用的时间，而 IDEA 选择的将大部分功能集成在一起，确确实实的方便了人们很多。单看 IDEA 的话，JetBrains 似乎并不怎么起眼，但是请不要忘记，它还有 PyCharm 等专门为其他语言设计的编辑器或编译器。

向上生长 三

发表于 2022-05-04 分类于 阅读笔记 ， 向上生长 本文字数： 1.8k 阅读时长 ≈ 3 分钟

        这是阅读《向上生长》后的第三篇阅读笔记。这一章也许是最难懂的一章吧，读了很多遍没有读懂这一章究竟是什么意思。不过既然九边在这一章中提到了工具，那我们不妨就来讨论一下工具对于我们的学习有哪些重要作用吧。

        九边将工具分为了两种——简单的工具与复杂的工具。简单的工具可以指木棍等简单的物品，复杂的工具就可以指 C4D、Blender、Adobe Photoshop、Ableton Live 11 还有之前有提到过的 Eclipse 与 JetBrains 的很多产品。无论简单还是复杂，它们都可以用来辅助人的创造。当然，只有工具是不够的，我们还需要使用工具的人。人在这里是一个很大的概念。

        动物为什么不能使用工具、不能使用语言、不能创造文字？那是因为没有人脑。所有的动物都有大脑，但并非所有的动物都有人脑。人脑是人类所特有的器官。人类所作出的任何动作都与人脑有关。所以若说的精细些，在使用工具时我们还需要人脑。它通过发出指令来使用工具，在使用工具的同时它又会将自己所学的记录在脑中。

        当然，有时候仅仅只有一个工具是不足的，我们还需要多个工具。以 Python 的爬虫为例，它不仅需要一个编辑代码的工具，还需要一个运行代码的工具；但只有这些是不行的，想要写好一个爬虫，它多多少少都需要用到一些“轮子”。“轮子”是什么？“轮子”就是“库”，它包含着一些码农已经写好的函数。

        爬虫可以分为三个步骤：请求、解析、存储，每个步骤都会用到不同的“轮子”。爬虫是模仿人类访问网页获取信息的一种工具。既然是模仿，那我们不妨想一下查找信息的几个步骤。我们不妨以找一张图片为例。首先我们需要与百度或者其他的搜索引擎建立 TCP 连接，TCP 链接建立完成后Web 浏览器会向 Web 服务器发送请求，然后 Web 服务器会给予 Web 浏览器答应，一般情况下 Web 服务器向浏览器发送了请求数据，他就会关闭 TCP 连接，此时便会在浏览器上显示我们所见到的网页了。（关于 TCP/IP 协议可以访问维基百科）第二步便是在网页上查找我们需要的图片，找到合适的图片后我们需要将它存储到本地计算机上。至此，一张图片便被我们成功的从网页上下载下来了。

        爬虫所做的，就是使用代码将上述行为模拟执行。在编辑代码是你需要一个编辑 Python 代码的工具，代码写完之后还需要一个执行代码的工具，你安装工具还需要一个系统环境。当然，也有一种工具可以执行代码也可以编写代码，比如 JetBrains 的 PyCharm 。关于编写的代码，在请求 HTTP 时，我们经常需要用到 urllib 、requests 等库；请求过后我们需要解析网页中我们所需要的信息，在解析网页信息时，我们经常需要用到 Beautifulsoup 、lxml 等库；提取了所需要的信息过后便需要将解析的信息存储下来，在存储所解析后的信息时，我们会用到 pymysql 。

        这时你会发现，两种不同的工具简易程度并不一样。自己从网站上保存一张图片只需要

短短几步，而自己编写网络爬虫则需要去了解一堆东西。比如什么是 TCP/IP 协议族 ，较为基础的 Python 语法，网页的数据结构等等。由此可见，简单的工具并不能带给我们多少突破，而复杂的工具将会教给我们很多东西。同时，工具的复杂程度不同，我们所需要使用工具的时间、理解工具原理的时间也就不同、工具的便捷程度也不相同。

        接下来便是时间，关于时间这方面我与九边的观点还是不大相同。时间长收获的东西并不一定就多，不断的投入没有回报的时间只是单纯的浪费而已。正如第二篇所说，你所做的仅仅只是读书，但没有“读书”。书不在多，精读则灵 ，读书的很多，但用心去读一本书的人很少。所投入的时间越长，所理解的可能就越深，但这一切的前提都是在这些时间上有一个总体的规划。在什么时间应该做什么，应该学会什么，这一点我想是十分重要的。

        一个宅在家里打游戏的阿宅和一些一天都坐在电脑前工作的程序员有什么相同的地方？没错，他们都不经常运动。还记得曾经看到的一个新闻，一个人在网吧连续打了几天游戏之后猝死。这是缺乏运动的表现。人的身体就如同一块电池，运动则是为这块电池充电。还记得曾经有人给我说过脑内运动也是运动。没错，大脑无时无刻在记录一些东西或者执行某条命令，这样看来大脑确实在运动，而且是无时无刻的运动。为什么会这样？因为它知道，当它不再运动的时候，它也就死了。不过大脑在运动只是为了能使“大脑”这个器官能活下去，心脏在跳动也只是为了“心脏”这个器官能够活下去，同样的，你在运动你也只是为了你自己能够活下去。

向上生长 二

发表于 2022-05-03 更新于 2022-05-04 分类于 阅读笔记 ， 向上生长 本文字数： 1.3k 阅读时长 ≈ 2 分钟

        这是阅读《向上生长》后的第二篇阅读笔记。究竟为什么要写这些看似没用的东西？也许是在借鉴作者“九边”的做法吧。他选择将收获的新知识写下来发一条微博，我则选择将阅读这本书每一章后所带给我的感觉记录在自己的 Blog 中。

        我并非与九边相同的前端工程师，对于前端的相关语言也只是略懂一二而已，不过这并不影响我写这篇阅读笔记的进程。“对之前读过的书有个记录，在吸收新知识的同时锻炼自己”。我从来没有想过还可以这样。对此，我真的要感谢九边教会我的这些。

        正如同九边所说，读书有用，他将读书比喻成了搜索引擎。这样的比喻很直观，很形象。搜索引擎只是工具，它是用来帮助人们学习而非取代人们学习。人就像是木匠，搜索引擎就是工具，而知识则相当于木材。只有工具而没有木材什么也做不成，只有木材没用工具也是什么也无法做到；只有工具和木材而没有木匠更是不必多说。

        又想起了曾经不知道在哪里听到的话，只要你想做，就不会缺少做这件事的时间。你有没有时间去读书纯粹是由你是否想去读书而决定的。因为你想去读，所以才会将时间用在这上面；如果你不想去读，那你怎么也不想将时间浪费在这上面。我与九边的观点是不同的，我认为没有人是懒惰的，每个人都是十分勤快的，他只是不想将时间浪费在这上面而已。而所谓浪费，则是你将时间用在了不想做的事情上。

        每个人都是不了解自己的，嘴上说着你不了解我之类的话，其实连自己也不了解自己。由这个思路来讲，其实每个人都不应该说“你并不了解我”之类的话。

        我与九边不一样，九边懂得要比我多得多。他会用神经学来解释习惯，而我只能用条件反射来解释。人在不适应某种习惯时会十分的难受，我认为这也是条件反射所导致的。他会用雪球理论来解释日积月累，而我只会用最简单的降雨来解释。

        在前面说过要注重数量，但其实单单注重数量是不行的。就拿读书来举例，你在一年中读完十本书，而我在一年中仅读完半本书，这有区别吗？有区别，且不单单时数量的区别。一年中读完10本书这确实符合上文所说的注重数量，但只在乎读书的数量就不去理解、消化书中所蕴含的知识和道理了吗？我认为这是不行的。他在一年中读完十本书但是并没有消化书中的知识，我在一年中只读完半本书但我将书中的知识消化完全了。读书的意义不在于读，而是在于理解。只有将书中的知识完全消化之后才可以被称之为读书。

        完全消化这在实际中并不可能，而我们所需要做的则是将书中的知识消化到 70% 及以上便可以了。

        在这里我似乎要反驳一下之前的观点了。以写小说为例子，有些人会因为打字速度太慢而放弃，如果按照九边作者所说，多写几年，打字速度一定会提升上去。其实这并非是结果，另一个结果可能是打字速度并没有提升多少。再结合上述读书的例子，这时我便对九边所说的话产生了怀疑。量的提升有可能并不会带来质的飞跃，由量变到质变的前提是从量变中学到些什么。打字速度由慢到快是自己在打字这方面上摸到了门道，在打字这一过程中学到了什么更为快速、简单的方法来提升自己的打字速度。

        换句话来总结上述则为在注重的同时还要在量变的过程中学到些什么技巧，否则你在这一过程中所做的纯属是浪费时间。

Minecraft的服务端种类很多，那么我们应该如何去选择并使用它们呢？

原版

Minecraft原版服务端。原版服务端又被称为官服，它的特性与原版相一致，是完全没有经过任何修改的，十分适合基友服联机使用，此外，它在很多地方都有方法下载，这其中包括MCWiki。

下载

下面将演示Minecraft 1.17.1原版服务端的下载方法。

首先我们在MCWiki打开MC Java 1.17.1的词条，此时我们将目光转向右边。

我们会发现下载处包含服务端，我们点击它即可下载。

你也可以在官网下载。注：官网默认为最新游戏版本。

Mod

Forge

Forge服务端，顾名思义，Forge服务端即为Forge所推出的服务端，可以使用Forge Mod。

下载

下面我将演示Minecraft 1.16.5 Forge服务端的下载方法。

首先我们打开Forge的官网，找到目前最新版本

点击Installer即可下载。

注意：我们现在下载的并非服务端，而是Forge的安装文件，部分版本的安装方法可能有些不同，我将会在最后进行讲解

双击运行我们下载的安装文件。

选择Install server

然后选择一个空文件夹，等待安装即可。

Forge服务端安装时间可能较长，其中可能需要手动补全libraries库，我们可以通过查看log来找到所需文件手动下载。建议使用特殊上网方法进行安装，此处不在过多赘述。

如果不想观看广告的话，可以使用如下方法：

Fabric

Fabric服务端，顾名思义，Fabric服务端即为Fabric所推出的服务端，可以使用Fabric Mod。

下载

下载Fabric服务端，同样需要进入Fabric的官网，我同样以目前最新版为例。

我们点击Download installer，无论是.jar后缀还是.exe后缀都不会影响我们接下来的操作。

点击Server，同样选择一个空目录等待安装即可。

如果需要更换Minecraft版本，请点击下拉框选择。

问题排查

通用

  1. 确认是否安装JDK
  2. 确认是否正确配置环境变量
  3. 确认服务端是否损坏
  4. 确认是否将eula协议设置为true
  5. 如是游戏版本为1.17+，请确认是否正确安装JDK 16
  6. 确认文件名与启动脚本中文件名一致
  7. 确认使用ansi编码保存
  8. 确认各参数之间添加空格

Forge服务端

  1. 确认是否补全libraries库
  2. 确认libraries库是否损坏

另外，端口被占用也会导致Minecraft服务器无法运行。

向上生长 一

发表于 2022-05-03 更新于 2022-05-04 分类于 阅读笔记 ， 向上生长 本文字数： 985 阅读时长 ≈ 2 分钟

        在2022的某一天，我意外的接触到了这本书——《向上生长》。从那时起，我便迷恋在这本书简洁的语言中。

        九边，这是个我从来没又听说过的名字。接触到他也仅仅是因为《向上生长》这本书。通过阅读这本书，我收获了很多。在书中，它教会了我如何去学习，学习的意义究竟是什么，我们为什么要学习。

        在书中，作者通过“热力学第二定律”解释了人为什么需要学习——获取持续输入的能量。不再学习的人就如同炉子里的火，没有能量的摄入结果只能是熄灭后化为灰烬。对于一个人来说，我们需要摄入的能量出了基本的营养外还有知识。我们可以通过挑战、学习来获取新的知识。

        知识可以分为两种，书本上存在的与书本上不存在或书本上不容易被注意、找到的。

        人生不能一路平坦，想要活下去我们必须要适应坎坷的道路，我们踏过这条坎坷的道路的过程可以理解为实践。不要怕，去挑战一下自己认为不可能的事情吧。这样才有新的知识摄入。

        但总是摄取自己所在的领域的知识是不够的，这样出去还是什么都不懂，因此我们需要摄取一些自己所在领域之外的知识。相信我，那总归不是无用的，总有一天他会用到的。

        有些读书的人会发现，自己长时间的阅读并没有收获太多，这其中的表现则可以形象的描述为自己的考试分数。不要担心，学习是一个循循渐进的过程，由量变从而产生质变。还记得在政治课本上的一句话：“量变是质变的基础，质变是量变的必然趋势”，有了量变才会在之后产生质变。一根木棍和一根铁棍的坚硬程度不同，这是因为“质”的不同。

        你可以将学习的知识比作一个水分子，一个水分子什么事情也无法做到，但成千上万乃至上亿个水分子呢？你吃东西的时候，食物所含有的能量会突然的传递给你吗？不会。因为吃进嘴里的食物有一个消化的过程。这并不是让你将书本上的字吃进肚子里让它消化，我相信也没有人会这样做。

        还记得小时候大人们常给我们讲过的有关于团结的故事吗？一根筷子很容易掰断，但一捆筷子则不容易被掰断。我们不妨想一下，从一根筷子到一捆筷子，这其中并没有“质”的变化，它仅仅是量的变化。团结又何尝不是如此？大家都是人，本质上并没有什么不同，所以团结的实质也只是量变所产生的结果而已。当你认为你的学习没有进展的时候，那请等一下吧，你所学习的知识会在之后的某一天突然“涌现”出来，这时，它们终于会派上了用场。当然，等待的过程并不是在亭子里坐着欣赏着自己也看不懂的东西，你可以在等待的时候学习新的知识。
